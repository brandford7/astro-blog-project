While the site now functions well, it might still lack visual polish. This is where proper styling comes in.

In this guide, we'll transform your functional FastHTML website into a visually stunning application using **Tailwind CSS** and **DaisyUI 5**. Tailwind is a utility-first CSS framework that allows rapid UI development through composable classes, while DaisyUI extends Tailwind with pre-built components—all without writing custom CSS. By the end of this tutorial, you'll have a professional-looking website that's not only functional but also aesthetically pleasing.

## Understanding Utility-First CSS with Tailwind

Before we dive in, let's understand why Tailwind CSS is revolutionary for developers, especially those coming from a Python background:

### Traditional CSS vs. Utility-First CSS

With traditional CSS, you typically:
1. Create custom class names
2. Write CSS rules in separate files
3. Switch between HTML and CSS files constantly
4. Manage growing stylesheets that become difficult to maintain

Tailwind's utility-first approach changes this workflow by:
1. Providing pre-defined utility classes
2. Allowing styling directly in your HTML/components
3. Eliminating the need to write and maintain custom CSS
4. Keeping styles localized to components (similar to FastHTML's philosophy)

For Python developers used to FastHTML's component-based approach, this is a natural fit!

### Step 1: Integrating Tailwind CSS and DaisyUI 5

First, we'll update our `components.py` file to include Tailwind CSS and DaisyUI via CDN. While a production site would use a build process for better performance, using CDNs is perfect for learning and development.

**File: `mywebsite/components.py`**

```python
from fasthtml.common import *

def header(current_page="/"):
    """
    Creates a consistent header with navigation using Tailwind and DaisyUI styling.

    Args:
        current_page: The current page path, used to highlight the active link

    Returns:
        A styled Header component with navigation
    """
    # Define the navigation links
    nav_items = [
        ("Home", "/"),
        ("About", "/about"),
        ("Contact", "/contact")
    ]

    # Create navigation items with appropriate styling
    nav_links = []
    for title, path in nav_items:
        # Apply special styling to the current page link
        is_current = current_page == path
        link_class = "btn btn-ghost normal-case text-lg"
        if is_current:
            link_class += " btn-active"

        nav_links.append(
            Li(
                A(title, href=path, cls=link_class)
            )
        )

    return Header(
        # DaisyUI navbar component
        Div(
            Div(
                # Website logo/name
                A("MyWebsite", href="/", cls="btn btn-ghost normal-case text-xl"),
                cls="navbar-start"
            ),
            # Navigation links in the center
            Div(
                Ul(
                    *nav_links,
                    cls="menu menu-horizontal px-1"
                ),
                cls="navbar-center"
            ),
            # Right side of navbar (could contain search, login, etc.)
            Div(
                # Theme toggle button using DaisyUI 5's theme controller
                Label(
                    Input(type="checkbox", cls="toggle theme-controller"),
                    cls="swap swap-rotate"
                ),
                cls="navbar-end"
            ),
            cls="navbar bg-base-100"
        ),
        cls="shadow-lg"
    )

def footer():
    """
    Creates a consistent footer with Tailwind and DaisyUI styling.

    Returns:
        A styled Footer component
    """
    current_year = 2025  # In a real app, use datetime to get current year

    return Footer(
        # DaisyUI footer component
        Div(
            Div(
                Div(
                    # Footer title
                    Span("MyWebsite", cls="footer-title"),
                    # Footer links
                    A("About us", href="/about", cls="link link-hover"),
                    A("Contact", href="/contact", cls="link link-hover"),
                    A("Jobs", href="#", cls="link link-hover"),
                    A("Press kit", href="#", cls="link link-hover"),
                    cls="flex flex-col"
                ),
                Div(
                    Span("Legal", cls="footer-title"),
                    A("Terms of use", href="#", cls="link link-hover"),
                    A("Privacy policy", href="#", cls="link link-hover"),
                    A("Cookie policy", href="#", cls="link link-hover"),
                    cls="flex flex-col"
                ),
                cls="footer p-10 bg-base-200 text-base-content"
            ),
            Div(
                P(f"© {current_year} MyWebsite - All rights reserved",
                  cls="text-center py-4 text-base-content"),
                cls="footer footer-center p-4 bg-base-300"
            )
        )
    )

def page_layout(title, content, current_page="/"):
    """
    Wraps page content with consistent header, footer, and styling.
    Integrates Tailwind CSS and DaisyUI 5.

    Args:
        title: The page title (appears in browser tab)
        content: The main content of the page
        current_page: The current page path for navigation highlighting

    Returns:
        A complete HTML page with header, content, and footer with Tailwind/DaisyUI styling
    """
    return Html(
        Head(
            Title(title),
            # Meta tags for better SEO and mobile display
            Meta(name="viewport", content="width=device-width, initial-scale=1.0"),
            Meta(name="description", content=f"{title} - MyWebsite built with FastHTML"),

            # Tailwind CSS
            Script(src="https://cdn.tailwindcss.com"),

            # DaisyUI CSS (which extends Tailwind)
            Link(href="https://cdn.jsdelivr.net/npm/daisyui@latest/dist/full.css", rel="stylesheet", type="text/css"),

            # Tailwind config script with DaisyUI 5 theming
            Script("""
                tailwind.config = {
                    theme: {
                        extend: {
                            colors: {
                                primary: "#4f46e5",
                                secondary: "#0ea5e9"
                            }
                        }
                    },
                    // DaisyUI 5 plugin configuration would normally go in CSS
                    // but we're simulating it here for the CDN approach
                    daisyui: {
                        themes: ["light", "dark"]
                    }
                }
            """, type="text/javascript")
        ),
        Body(
            # data-theme attribute enables DaisyUI theming
            # We start with the light theme but users can toggle it
            Div(
                # Include header with current page highlighted
                header(current_page),

                # Main content area
                Main(
                    content,
                    cls="min-h-screen p-4"
                ),

                # Include footer
                footer(),

                # Apply DaisyUI theming
                data_theme="light"
            )
        )
    )
```

In this updated `components.py`:

1. **Header**: We've transformed the basic header into a DaisyUI navbar with:
   - A brand name in the navbar-start section
   - Navigation links in the navbar-center
   - A theme toggle switch in the navbar-end using DaisyUI 5's theme controller
   - Highlighting for the current page

2. **Footer**: We've created a two-part footer using DaisyUI's footer component:
   - Main section with links organized in columns
   - Bottom section with copyright information

3. **Page Layout**: We've integrated:
   - Tailwind CSS via CDN
   - DaisyUI 5 via CDN (using the latest beta version)
   - A Tailwind configuration script for theme customization
   - A theme toggle functionality
   - Responsive container for the main content

## Understanding DaisyUI 5 Core Concepts

DaisyUI 5 brings several improvements over previous versions. Here are the key concepts you should understand:

### Theme Management in DaisyUI 5

DaisyUI 5 introduces a new way to manage themes through CSS files. While we're using CDN for simplicity, in a production environment you would define themes like this:

```css
@import "tailwindcss";
@plugin "daisyui" {
  themes: light --default, dark --prefersdark;
}
```

This new syntax allows you to:
- Specify which themes to include (reducing bundle size)
- Set default themes with `--default` flag
- Set preferred dark themes with `--prefersdark` flag

### Creating Custom Themes in DaisyUI 5

DaisyUI 5 uses CSS custom properties for theme colors, allowing for more flexibility and better color science with OKLCH colors:

```css
@plugin "daisyui/theme" {
  name: "mytheme";
  default: true;

  --color-primary: oklch(55% 0.3 240);
  --color-secondary: oklch(70% 0.25 200);
  --color-accent: oklch(65% 0.25 160);
  --color-neutral: oklch(50% 0.05 240);

  /* border radius */
  --radius-selector: 1rem;
  --radius-field: 0.25rem;
}
```

## Utility-First Approach with Tailwind

Tailwind CSS takes a utility-first approach to styling, providing low-level utility classes that you can combine to build custom designs directly in your FastHTML components. This approach offers several advantages:

1. **No CSS Files**: You don't need to write or maintain separate CSS files
2. **No Naming Conventions**: No need to invent class names for every element
3. **Rapid Development**: Make changes directly in your components without switching files
4. **Responsive Design**: Built-in responsive modifiers like `md:` and `lg:`

### Common Utility Categories

Here are some common categories of utility classes you'll use in Tailwind:

#### Typography

- `text-{size}`: Font size (e.g., `text-sm`, `text-lg`, `text-2xl`)
- `font-{weight}`: Font weight (e.g., `font-normal`, `font-bold`)
- `text-{color}`: Text color (e.g., `text-blue-500`, `text-gray-800`)
- `text-{alignment}`: Text alignment (e.g., `text-center`, `text-left`)

#### Spacing

- `m-{size}`, `mx-{size}`, `my-{size}`, `mt-{size}`, etc.: Margin
- `p-{size}`, `px-{size}`, `py-{size}`, `pt-{size}`, etc.: Padding

#### Layout

- `flex`, `grid`: Display mode
- `items-{position}`: Align items (e.g., `items-center`)
- `justify-{position}`: Justify content (e.g., `justify-between`)
- `w-{size}`, `h-{size}`: Width and height

#### Backgrounds and Borders

- `bg-{color}`: Background color (e.g., `bg-blue-500`)
- `border`, `border-{size}`: Border width
- `rounded`, `rounded-{size}`: Border radius

#### Responsive Design

- `{breakpoint}:{utility}`: Apply utility at a specific breakpoint (e.g., `md:flex-row`)
- Common breakpoints: `sm`, `md`, `lg`, `xl`, `2xl`

## Common Patterns and Best Practices

As you work with Tailwind CSS and DaisyUI 5 in FastHTML, keep these best practices in mind:

### 1. Organize Related Styles

Group related Tailwind classes together for readability:

```python
# Instead of this (mixed order)
cls="text-lg font-bold m-2 p-4 text-blue-500 rounded"

# Do this (organized by category)
cls="text-lg font-bold text-blue-500 p-4 m-2 rounded"
```

### 2. Extract Repeated Patterns

For repeated style patterns, consider creating helper functions:

```python
def primary_button(text, href="#"):
    return A(text, href=href, cls="btn btn-primary")

def section_heading(text):
    return H2(text, cls="text-3xl font-bold text-center mb-8")
```

### 3. Use Responsive Prefixes Consistently

Be consistent with your responsive design approach:

```python
# Mobile-first approach (recommended)
cls="block md:flex"  # Stack on mobile, row on desktop

# Desktop-first approach
cls="flex sm:block"  # Row on desktop, stack on mobile
```

### 4. Keep Accessibility in Mind

Use proper semantic HTML and ARIA attributes alongside your styling:

```python
Button("Close",
       cls="btn btn-sm btn-circle",
       aria_label="Close")
```
